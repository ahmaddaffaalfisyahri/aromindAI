Q: "Menggunakan ML model apa?"
Jawaban: Sistem ini menggunakan Rule-Based Scoring Algorithm, bukan Machine Learning tradisional (neural network/decision tree).

Alasan pemilihan:
  • Lebih interpretable - bisa dijelaskan mengapa parfum X direkomendasikan
  • Tidak butuh training data besar (dataset hanya ~100 parfum)
  • Mudah di-tuning dan diperbaiki
Untuk fitur OCR: Menggunakan Gemini 2.0 Flash (Google Generative AI) sebagai Vision Language Model untuk membaca teks dari gambar parfum.
Q: "Kenapa bisa munculin parfum dari cuaca? Di dataset kan gak ada kolom cuaca?"
Jawaban: Benar, dataset TIDAK memiliki kolom cuaca. Yang dilakukan adalah MAPPING cuaca ke aroma notes yang sudah ada di dataset.

Logikanya:
  • Cuaca PANAS ‚Üí cari parfum dengan notes: citrus, fresh, aquatic
  • Cuaca HUJAN ‚Üí cari parfum dengan notes: woody, warm, amber
Implementasi: Di main.py baris 82-95, cuaca di-convert menjadi target_tags, lalu tags ini dicocokkan dengan kolom "top notes", "mid notes", "base notes" di dataset.

Dasar teori: Berdasarkan prinsip parfumeri bahwa aroma tertentu lebih cocok di cuaca tertentu.
Q: "Manggil backend-nya dimana?"
Jawaban: Backend dipanggil dari file aromind_api.dart

Flow lengkap:
  1. User klik tombol "Cari parfum terbaik" di main.dart baris 147
  2. Memanggil api.getRecommendations()
  3. Di aromind_api.dart baris 24, request POST dikirim ke Railway
  4. Backend memproses dan mengembalikan JSON
  5. Response di-parse menjadi object Perfume dan ditampilkan
URL Backend: aromind_api.dart baris 13:
https://aromindbackend-production.up.railway.app
Q: "Match score untuk memunculkan hasil parfum gimana?"
Jawaban: Setiap parfum dihitung skor-nya:
Kriteria Poin Penjelasan
Tag aroma cocok +2 per tag Jika notes parfum mengandung target tag (misal "citrus")
Budget sesuai +1 Jika harga dalam range budget_min - budget_max
Harga terlalu mahal -1 Jika harga > 1.5x budget_max
Hasil: Parfum dengan skor > 0 masuk list, diurutkan dari skor tertinggi, ambil 10 teratas.
Lokasi kode: main.py baris 109-138

2. ALGORITMA SCORING - Step by Step

üî¢ Rumus: Skor = (Tag Match √ó 2) + Budget Match (1) - Penalti (1)

Step File Baris Proses
1 main.py 109-113 Gabungkan notes: top + mid + base (lowercase)
2 main.py 115-117 Cek tag match: if tag in notes: score += 2
3 main.py 119-127 Cek budget: if budget_min <= price <= budget_max: score += 1
4 main.py 125-127 Penalti: if price > budget_max * 1.5: score -= 1
5 main.py 129-130 Filter: Hanya score > 0 yang masuk
6 main.py 133 Urutkan: sorted(..., reverse=True)
7 main.py 136 Ambil top 10: results[:10]

4. Sistem Deteksi Kategori Aroma (Fresh, Woody, Sweet, Floral)

üéØ Bagaimana sistem mendeteksi parfum cocok untuk kategori tertentu?

Backend menggunakan keyword matching dari kolom "notes" di dataset untuk menentukan kategori parfum:

Kategori Keywords yang Dicari Contoh Notes
Fresh citrus, lemon, bergamot, lime, orange, grapefruit, aquatic, marine, fresh, green, mint "fresh citrus, bergamot, mint"
Woody woody, wood, cedar, sandalwood, oud, vetiver, patchouli, oakmoss, pine, birch, amber "woody amber, cedar, patchouli"
Sweet vanilla, sweet, caramel, honey, sugar, gourmand, toffee, candy, chocolate "sweet vanilla, caramel, honey"
Floral rose, jasmine, floral, flower, lily, peony, tuberose, iris, violet, magnolia, orchid "jasmine, rose petals, tuberose"

Logika: Jika user memilih preferensi "Woody", backend mencari parfum yang notes-nya mengandung keyword woody.

Kode: main.py baris 82-95 (mapping preference ‚Üí target_tags)

5. Cara Mendapatkan Skor Tertinggi

üèÜ Parfum dengan skor tertinggi adalah yang paling BANYAK cocok dengan kriteria user

Faktor yang mempengaruhi skor:

Faktor Poin Cara Memaksimalkan
Tag Match (Preferensi) +2 per tag Parfum dengan banyak keyword yang match akan dapat skor lebih tinggi
Tag Match (Cuaca) +2 per tag Parfum yang cocok dengan cuaca (misal: citrus untuk panas) dapat bonus
Tag Match (Aktivitas) +2 per tag Parfum yang cocok dengan aktivitas (misal: clean untuk kerja) dapat bonus
Budget Sesuai +1 Harga parfum dalam range budget user
Penalti Harga -1 Jika harga > 1.5x budget max, parfum kehilangan poin

Input: Preferensi=Woody, Cuaca=Hujan, Aktivitas=Date, Budget=100k-300k

Target Tags: woody, warm, amber, sweet, romantic, sensual

  • Notes parfum: "woody amber, warm vanilla, sweet musk, romantic rose"
  • Tag "woody" ‚Üí +2
  • Tag "amber" ‚Üí +2
  • Tag "warm" ‚Üí +2
  • Tag "sweet" ‚Üí +2
  • Harga 200.000 dalam budget ‚Üí +1
  • Total Skor: 9 poin üèÜ

Kode: main.py baris 109-138
